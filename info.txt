В чому полягає різниця між Spring Framework та Spring Boot?

Spring Framework — це фреймворк для розробки Java-додатків, який надає підтримку для різних аспектів, таких як ін'єкція залежностей, транзакційне управління, безпека тощо.
Spring Boot — це розширення Spring Framework, яке спрощує процес налаштування та розгортання додатків. Воно автоматично конфігурує ваш додаток на основі залежностей, які ви вказали, і надає вбудовані сервери, такі як Tomcat, для швидкого запуску.
Для чого в структурі проекту потрібен файл pom.xml?

Файл pom.xml використовується для управління залежностями та конфігурації проекту в Maven. Він визначає, які бібліотеки необхідно завантажити, версії пакетів, плагіни для збірки та інші налаштування.
Для чого потрібна анотація @SpringBootApplication? Що буде, якщо її прибрати?

Анотація @SpringBootApplication є комбінацією трьох інших анотацій: @Configuration, @EnableAutoConfiguration і @ComponentScan. Вона використовується для позначення основного класу Spring Boot-додатку. Якщо її прибрати, вам доведеться вручну налаштувати багато аспектів, які автоматично конфігуруються Spring Boot.
Для чого потрібен інтерфейс CommandLineRunner?

Інтерфейс CommandLineRunner використовується для виконання коду після того, як Spring Boot-додаток був ініціалізований. Метод run цього інтерфейсу буде автоматично викликаний Spring Boot після завантаження додатку.
Чим інтерфейс CommandLineRunner відрізняється від ApplicationRunner?

Обидва інтерфейси використовуються для виконання коду після запуску додатку, але вони відрізняються типом аргументів, які передаються в метод run. У CommandLineRunner передаються звичайні рядкові аргументи (String... args), тоді як у ApplicationRunner передається об'єкт ApplicationArguments, який надає більше можливостей для роботи з аргументами командного рядка.
